修复 telegram_bot.py 中 send_message 不走代理的问题。

问题：_cmd_start 等方法中用 self.bot.send_message() 发送消息，但 self.bot 不带代理配置，导致 send_message 超时。

修复方案：
1. 添加 _send_message 方法，用 self.http_session 直接 POST 到 Telegram API
2. 将所有 cmd_* 方法中的 self.bot.send_message() 替换为 self._send_message()

_send_message 方法：
```python
async def _send_message(self, chat_id: int, text: str):
    """用 http_session 发送 Telegram 消息（走代理）"""
    url = f"https://api.telegram.org/bot{self.token}/sendMessage"
    async with self.http_session.post(url, json={"chat_id": chat_id, "text": text}) as resp:
        data = await resp.json()
        if not data.get("ok"):
            self._logger.error(f"send_message failed: {data}")
```

修改位置：找到所有 self.bot.send_message( 出现的地方（约6-7处），全部替换为 await self._send_message(。

发送长消息（>4096字符）时分段发送，每段调用 _send_message。
