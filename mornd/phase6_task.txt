# Phase 6: AES-256 数据加密（~1天）
## 目标
为 SQLite 存储的记忆数据添加 AES-256 传输中加密保护。

## 设计原则
- 不要改数据库结构（向后兼容）
- 不要加密 FTS5 索引（否则搜索完全不可用）
- 只加密 `description` 字段（实际对话内容）
- 搜索降级：FTS5 继续在加密文本上搜索（精确匹配 token），LIKE 搜索不再支持中文全文搜索，改为解密后内存搜索
- 现有数据平滑迁移：检测旧数据（纯文本 JSON），读取时自动解密

## 改动文件
1. morn_core/memory/crypto.py — 新增，加密/解密抽象
2. morn_core/memory/store.py — 修改，写入时加密，读取时解密
3. tests/test_crypto.py — 新增测试

## 实现细节

### 1. crypto.py

```python
import os
import json
import base64
import logging
from pathlib import Path

from Crypto.Cipher import AES
from Crypto.Protocol.KDF import PBKDF2


class MemoryCrypto:
    """AES-256-GCM 内存加密。
    
    密钥从文件读取或自动生成。
    key 文件路径：{data_dir}/memory.key（chmod 600）
    
    加密格式：base64(nonce + ciphertext + tag)
    解密：reverse
    
    使用 PBKDF2 派生密钥（32字节）以兼容各种长度的密钥种子。
    """
    
    KEY_SIZE = 32  # AES-256
    SALT_SIZE = 16
    NONCE_SIZE = 12  # GCM 推荐
    TAG_SIZE = 16
    
    def __init__(self, data_dir: Path):
        self.data_dir = Path(data_dir)
        self._key = None
        self._load_or_create_key()
    
    def _load_or_create_key(self):
        key_path = self.data_dir / "memory.key"
        if key_path.exists():
            raw = key_path.read_bytes()
            # 支持两种格式：32字节原始密钥，或 base64 编码的密钥
            if len(raw) == self.KEY_SIZE:
                self._key = raw
            else:
                try:
                    self._key = base64.b64decode(raw.strip())
                    if len(self._key) != self.KEY_SIZE:
                        raise ValueError(f"key size mismatch: {len(self._key)}")
                except Exception:
                    raise RuntimeError("memory.key 格式错误")
        else:
            self._key = os.urandom(self.KEY_SIZE)
            key_path.write_bytes(self._key)
            key_path.chmod(0o600)
    
    def encrypt(self, plaintext: str) -> str:
        """加密字符串，返回 base64 编码的密文。"""
        if not plaintext:
            return plaintext
        nonce = os.urandom(self.NONCE_SIZE)
        cipher = AES.new(self._key, AES.MODE_GCM, nonce=nonce)
        ciphertext, tag = cipher.encrypt_and_digest(plaintext.encode("utf-8"))
        # 格式：nonce + ciphertext + tag
        payload = nonce + ciphertext + tag
        return "ENC:" + base64.b64encode(payload).decode("ascii")
    
    def decrypt(self, encrypted: str) -> str:
        """解密 base64 编码的密文，返回明文。失败时返回原始字符串。"""
        if not encrypted:
            return encrypted
        if not encrypted.startswith("ENC:"):
            return encrypted  # 未加密的数据（向后兼容）
        try:
            payload = base64.b64decode(encrypted[4:])
            nonce = payload[:self.NONCE_SIZE]
            tag = payload[-self.TAG_SIZE:]
            ciphertext = payload[self.NONCE_SIZE:-self.TAG_SIZE]
            cipher = AES.new(self._key, AES.MODE_GCM, nonce=nonce)
            return cipher.decrypt_and_verify(ciphertext, tag).decode("utf-8")
        except Exception as e:
            logger = logging.getLogger("morn.crypto")
            logger.error(f"decrypt failed: {e}")
            return encrypted  # 降级：返回原始数据
    
    def is_encrypted(self, text: str) -> bool:
        return bool(text) and text.startswith("ENC:")
```

**注意：** 使用 `pycryptodome`（已在 pyproject.toml 中声明依赖）。
需要确保依赖已安装：`pip install pycryptodome`

### 2. 修改 MemoryStore

#### a) __init__ 新增加密参数
```python
def __init__(self, data_dir: Path, enable_encryption: bool = False):
    ...
    self._crypto = MemoryCrypto(data_dir) if enable_encryption else None
```

#### b) add_capsule 中加密 description
```python
async def add_capsule(self, capsule: dict) -> str:
    ...
    # 在拼装 data 字典后、写入 SQLite 之前，加密 description
    if self._crypto and data["description"]:
        data["description"] = self._crypto.encrypt(data["description"])
    ...
```

#### c) 所有读方法中解密 description
所有从 SELECT 返回 capsule 的地方（_row_to_dict 或类似 helper），在返回前解密 description：
```python
def _decrypt_capsule(self, capsule: dict) -> dict:
    """解密 capsule 中的加密字段"""
    if capsule and self._crypto:
        capsule = dict(capsule)
        capsule["description"] = self._crypto.decrypt(capsule.get("description", ""))
    return capsule
```

在 search_fts、get_recent、get_capsule、search_by_entity、search_by_timerange 等所有返回 capsule list/dict 的方法中，对每条结果调用 _decrypt_capsule。

#### d) search_fts 中 LIKE 搜索的降级
当前 search_fts 有两个阶段：FTS5 MATCH 和 LIKE 兜底。
加密后 FTS5 仍然工作（索引的是加密文本，但精确 token 匹配对加密内容也成立）。
LIKE 兜底不工作了（加密后的文本 NOT LIKE 用户查询的文字）。

修改：LIKE 搜索降级为全表扫描 → 解密所有 description → 在 Python 中做 `query in description` 匹配。

```python
# 2. 中文 LIKE → 解密后内存搜索（加密降级）
has_chinese = any(ord(c) > 127 for c in query)
if self._crypto or has_chinese or len(results) == 0:
    try:
        cursor = await self.db.execute(
            "SELECT * FROM capsules WHERE forget_creator = 0 ORDER BY timestamp DESC LIMIT ?",
            (limit * 2,))
        for row in await cursor.fetchall():
            d = self._decrypt_capsule(dict(row))
            desc = d.get("description", "")
            if query.lower() in desc.lower():
                if d["event_id"] not in seen_ids:
                    results.append(d)
                    seen_ids.add(d["event_id"])
    except Exception:
        pass
```
