# Phase 9: 本地模式可用性检测
## 问题
ConfigManager 允许用户通过自然语言切换到纯本地模式（local），但本地 Ollama 服务可能未运行。切换后 ChatEngine._call_local 会失败，返回降级消息 "我现在有点连接不上"。

## 改动
在 ConfigManager.detect_and_apply 中，对切换到 local 模式的请求增加可用性检测。

## 实现

### 修改 morn_core/chat/engine.py

在 ConfigManager 类中增加方法 `_local_model_available() -> bool`：

```python
async def _local_model_available(self) -> bool:
    """检查本地 Ollama 服务是否可用。"""
    import aiohttp
    host = self.config.get("ollama_host", "http://localhost:11434")
    try:
        async with aiohttp.ClientSession() as session:
            async with session.get(f"{host}/api/tags", timeout=3) as resp:
                if resp.status == 200:
                    data = await resp.json()
                    models = data.get("models", [])
                    return len(models) > 0
                return False
    except Exception:
        return False
```

将 `detect_and_apply` 改为 async 方法（当前是同步方法，但 ChatEngine.chat 已经是 async，调用处 await 即可）。

在模式切换的代码块中，对 local 模式增加检测：
```python
for keyword, mode in mode_map.items():
    if keyword in text_lower and self._is_config_request(text_lower, keyword):
        if mode == "local" and not await self._local_model_available():
            return True, "本地模型没有在运行。先启动 Ollama 再切本地模式，或者先用云端模式继续聊。"
        ...
```

### 修改 tests/test_nl_config.py

现有的测试中 `test_detect_mode_switch_local` 调用 `cm.detect_and_apply("改成纯本地模式")`，这是一个 async 方法。但测试中 ConfigManager 传入的 engine=None，所以 `_local_model_available` 会尝试 http 请求。

解决办法：
1. 让 `_local_model_available` 在 `self.engine is None` 时返回 True（测试模式跳过检测）
2. 添加一个新的测试 `test_local_switch_rejected_when_ollama_down`

```python
async def test_local_switch_rejected_when_ollama_down(self):
    """当 Ollama 不可用时，切换到本地模式应被拒绝并提示。"""
    cm = ConfigManager(Path("/tmp/test_config.json"), {"mode": "hybrid", "ollama_host": "http://localhost:11434", "ollama_model": "qwen2.5:1.5b"}, None)
    # engine=None 时 _local_model_available 检查真实 HTTP
    # 如果本地确实没有 ollama，它会返回 False
    matched, reply = await cm.detect_and_apply("改成纯本地模式")
    if matched:
        # 如果匹配了，回复应该是提示信息而非成功切换
        assert "本地模型没有在运行" in reply or "正在运行" in reply
```
