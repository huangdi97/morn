修复 telegram_bot.py 两个问题：

问题1：self.bot 使用了错误的 session 类型（aiohttp.ClientSession 而非 aiogram 的 session），导致 send_message 出错 "'ClientSession' object is not callable"
问题2：dispatch 中 send_message 出错后 poll error 被捕获，但 offset 没有更新，导致同一个 update 被反复处理

修复方案：
1. 完全移除 aiogram Bot，改用 http_session 直接 POST 发送消息
2. 添加 _send_message 方法
3. 所有 cmd_* 方法改为调用 _send_message
4. start() 中 offset 在 dispatch 之前更新

具体修改 morn_core/presence/telegram_bot.py：

1. 删除导入：from aiogram import Bot、from aiogram.types import Message、from aiogram.client.default import DefaultBotProperties

2. 保留 from aiohttp import ClientSession

3. __init__ 中删除 self.bot = Bot(...) 这一行

4. 添加 _send_message 方法：
   async def _send_message(self, chat_id: int, text: str):
       url = f"https://api.telegram.org/bot{self.token}/sendMessage"
       async with self.http_session.post(url, json={"chat_id": chat_id, "text": text}) as resp:
           data = await resp.json()
           if not data.get("ok"):
               self._logger.error(f"send_message failed: {data}")

5. 所有 cmd_* 方法中的 self.bot.send_message( 替换为 await self._send_message(

6. start() 方法中，offset 在 dispatch 前更新：
   for update in data.get("result", []):
       update_id = update["update_id"]
       if "message" in update:
           try:
               await self._dispatch(update["message"])
           except Exception as e:
               self._logger.error(f"dispatch error: {e}")
       self._polling_offset = update_id + 1

7. stop() 方法中删除 await self.bot.session.close()
